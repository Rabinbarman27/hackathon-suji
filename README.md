# StudyHub (migrated to React + Vite)

This repo is a migration of the original static site into a small React app bootstrapped with Vite.

Quick commands

Install dependencies:
```bash
npm install
```

Run dev server:
```bash
npm run dev
```

Build production:
```bash
npm run build
```

Preview production build locally:
```bash
npm run preview
```

Deployment

- Vercel: the repo includes `vercel.json` configured to rewrite all routes to `index.html` for SPA routing. Set the build command to `npm run build` and the output directory to `dist` on Vercel.

Notes

- Local data (accounts, progress) is stored in `localStorage` for demo purposes.
- Before committing again, avoid committing `node_modules` and `dist` (see `.gitignore`).

### ğŸ”‘ Key Concepts Explained

#### 1. Authentication (Mock)
We use a simple "mock" authentication system. Instead of talking to a server, we save the user's credentials in the browser's `localStorage`.

*See `script.js`*:
```javascript
// Saving a user
const user = { email, password };
localStorage.setItem("studentUser", JSON.stringify(user));

// Checking login
const savedUser = JSON.parse(localStorage.getItem("studentUser"));
if (inputEmail === savedUser.email) { ... }
```
**Challenge for You**: Try adding a "Forgot Password" feature!

#### 2. Data Persistence (LocalStorage)
The app feels like it remembers you because we use `localStorage`. This is a key-value store in your browser.

*   `isLoggedIn`: "true" or "false".
*   `points`: Integer number of points earned.
*   `studyProgress`: A JSON object tracking minutes studied by date and mode.

*See `progress.js` (Line 41)*:
```javascript
function saveProgress(minutes) {
  let progress = JSON.parse(localStorage.getItem("studyProgress")) || {};
  // ... update logic ...
  localStorage.setItem("studyProgress", JSON.stringify(progress));
}
```

#### 3. Navigation
Since this is a multi-page site (not a Single Page App), navigation is handled by standard HTML links or JavaScript redirection.

```javascript
window.location.href = "dashboard.html"; // Redirects user
```

### ğŸ› Known Limitations (Good First Issues!)
This project is a perfect starting point. Here are things *you* can fix or improve:

1.  **Ephemeral Notes**: The logic in `notes.js` appends notes to the screen but *doesn't save them* to `localStorage`. If you refresh `notes.html`, your notes disappear! **Task**: Implement `saveNotes` using `localStorage` so they persist.
2.  **Security**: The passwords are stored in plain text in the browser. In a real app, you would never do this!
3.  **Chart Library**: We use Chart.js, but make sure the script tag is properly included in all pages that need it.
